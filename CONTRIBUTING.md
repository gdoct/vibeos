# Contributing

Thank you for contributing to VibeOS.

## Development Workflow

- All implementation must be produced by a premium coding agent (for example: Claude Opus or OpenAI Codex).
- No manual coding in source files.
- Changes are made by prompting the coding agent only.

## Branch Rules

- When creating a branch for new work, add the planned features for that work to `ROADMAP.md`.
- Keep roadmap updates clear and scoped to the branch purpose.

## Coding Standards

- Follow strict, modern C style for all code changes.
- Keep code simple, explicit, and portable.
- Adhere to existing project conventions and standards in the repository.

## Refactoring

Don't be afraid to refactor at large, but if you do, make sure to test extensively and update documentation as needed.
